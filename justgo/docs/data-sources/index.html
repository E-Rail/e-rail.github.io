01 / VERIFIED SNAPSHOT

The totals reconcile.

The catalog has 58 cities. Forty-six have included OpenStreetMap network baselines and 12 are catalog-only. Two of the 46 baseline cities also have included station packs.

58 cataloged cities = 46 with included OSM network baselines + 12 catalog-only 46 baseline cities = 2 with included city packs + 44 geometry-only, source-pending 2 included city packs = 444 Beijing station records + 162 Hong Kong station records = 606 station records 56 source-pending cities = 44 geometry-only + 12 catalog-only
What “source-pending” means

The project has not included a supported station-enrichment pack for that city. It does not mean the city or its operator has no public information; it means that information is not included in this build as a supported JustGo dataset.

02 / COUNTING RULES

Four labels prevent double-counting.

LabelExact meaningCounts as offline app content?Can trigger a network request?
IncludedData or media delivered as part of the app build.YESNot merely to read the included item.
External referenceA stored URL or identifier pointing to a provider-controlled operator page or layout.NOOnly after an explicit user action opens the destination.
Live API referenceA station or line reference that can request a current value from an online provider.THE REFERENCE ONLYYes, when current results are requested.
Personal mediaA rider-selected photo or file in the app’s private device layer.PRIVATE LOCAL DATANot for routing or guidance; device services may govern the rider’s original media selection.

An external reference can be present in an included station record without making the destination page or layout an included asset. Likewise, a live-arrival reference can be included while the current arrival result remains online-only.

06 / EXTERNAL & LIVE BEHAVIOR

User action and network behavior are explicit.

01Included station view

Read the included city-pack record locally. No operator webpage is contacted merely because a reference exists.

02External layout

Open the destination only after the rider taps it. Do not download, cache, or prefetch the page or count it as offline content.

03Live arrivals

When the rider requests current Hong Kong arrivals, contact the DATA.GOV.HK government transport API using the referenced station and line context.

04Personal media

Show private on-device context only to the rider. Never use it to select a route, calculate guidance, identify a door, or infer a transfer.

08 / MAINTENANCE RULES

Future updates must preserve the distinctions.

  • Increase an included count only when the corresponding data ships as part of the app content.
  • Count a layout as external unless a licensed layout file itself is included.
  • Count a live reference separately from a static schedule and from a live response.
  • Do not count a transfer as verified without a source-backed indoor path that establishes the transfer.
  • Do not use personal station media as source evidence for public routing or guidance.
  • Update the privacy, terms, attribution, services, download, product, and documentation pages together when a data boundary changes.
