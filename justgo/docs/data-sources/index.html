01 / VERIFIED SNAPSHOT

The totals reconcile.

Round 35 catalogs 58 cities: 46 have route-picker baselines and 12 are catalog-only. Two baseline cities also have included offline station packs.

58 cataloged cities = 46 with route-picker baselines + 12 catalog-only 46 route-picker cities = 2 with included offline city packs + 44 geometry-only, source-pending 2 included offline city packs = 444 Beijing station records + 162 Hong Kong station records = 606 station records 56 source-pending cities = 44 geometry-only + 12 catalog-only
What “source-pending” means

The project has not included a supported station-enrichment pack for that city. It does not mean the city or its operator has no public information; it means that information is not included in this build as a supported JustGo dataset.

02 / COUNTING RULES

Four labels prevent double-counting.

LabelExact meaningCounts as offline app content?Can trigger a network request?
IncludedData or media delivered as part of the app build.YESNot merely to read the included item.
Official-resource hyperlinkReviewed URL metadata for a provider-controlled page or file. The destination content is not included.NOOnly after the rider taps the link.
Live API referenceA station or line reference that can request a current value from an online provider.THE REFERENCE ONLYYes, when current results are requested.
Personal mediaA rider-selected photo or file in the app’s private device layer.PRIVATE LOCAL DATANever used as verified routing, transfer, platform, or door data.

A hyperlink record can ship in the directory without making its destination an included asset. Likewise, a live-arrival reference can be included while the current arrival result remains online-only.

03 / OFFICIAL-RESOURCE DIRECTORY

330 records, 275 unique targets.

The directory is separate from the two offline city packs. It stores reviewed factual URL metadata; it does not add station data or offline map coverage to a city.

URL metadata is not operator content

Operator pages and files open only after the rider taps a link. JustGo does not bundle, fetch, preview, cache, prefetch, or redistribute them.

07 / EXTERNAL & LIVE BEHAVIOR

User action and network behavior are explicit.

01Included station view

Read the included city-pack record locally. No operator webpage is contacted merely because a reference exists.

02Official-resource link

Open the provider destination only after the rider taps it. JustGo does not bundle, fetch, preview, cache, prefetch, or redistribute the page or file.

03Live arrivals

When the rider requests current Hong Kong arrivals, contact the DATA.GOV.HK government transport API using the referenced station and line context.

04Personal media

Keep it entirely on-device. Never use it as verified routing, transfer, platform, or door data.

09 / MAINTENANCE RULES

Future updates must preserve the distinctions.

  • Increase an included count only when the corresponding data ships as part of the app content.
  • Count directory records separately from unique targets and from included files.
  • Count an operator map as external unless a licensed map file itself is included.
  • Count a live reference separately from a static schedule and from a live response.
  • Do not count a transfer as verified without a source-backed indoor path that establishes the transfer.
  • Do not use personal station media as source evidence for public routing or guidance.
  • Update the privacy, terms, attribution, services, download, product, and documentation pages together when a data boundary changes.
