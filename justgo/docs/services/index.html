03 / LOCATION & PLACES

Location is requested for a reason, then stopped.

LocationService

Wraps CLLocationManager and exposes the current fix, authorization state, and a single asynchronous location request.

ACCURACY TARGETkCLLocationAccuracyBest, 10 m distance filter.
CACHE FAST PATHAuthorized fix, at most 30 seconds old and at most 100 m horizontal uncertainty.
REQUEST TIMEOUT15 seconds before pending callers receive “current location unavailable.”
BATTERY BEHAVIORContinuous updates run only while a request is pending and stop on success, failure, timeout, or cancellation.
PREWARMOne-shot requestLocation(); does nothing and shows no prompt unless already authorized.
PERMISSIONWhen-in-use is requested lazily. Denied or restricted state clears the in-memory fix.

MapKitPlaceSearchProvider

Uses MKLocalSearch for addresses and POIs and creates a fresh CLGeocoder for each reverse-geocode call so overlapping requests do not cancel one another.

RESULT TYPESAddress and point-of-interest.
SEARCH BIASVisible map region when available; otherwise an 80 km × 80 km region around the selected city.
RESULT LIMITS12 for map place search, 20 for combined station search, 6 for suggestions.
DEADLINE12-second race around MapKit calls so UI spinners can recover from a stalled framework request.
CANCELLATIONCallers guard against stale results because MapKit requests may continue after Swift task cancellation.
FAILURESurfaces a user-facing “place search requires a network connection” result.

04 / METRO GRAPH

The subway path is local.

BundledMetroNetworkService

Loads one of 46 bundled JSON files containing city bounds, stations, lines, service patterns, and physical-track polylines.

VALIDATIONCity ID must match the requested ID and geometryKind must equal physicalTrack.
CHEAP CITY MATCHDecodes bounds-only summaries for all cities before loading full network objects.
DECODE THREADFile reads and JSON decoding run in a detached utility task.
CACHESFull networks, display stations, summaries, and normalized station-name indexes.
MAP RULESNormal station markers appear only at closer zoom; transfer stations remain visible farther out.
RELEASEMemory warning clears full networks, display stations, and normalized indexes.

BundledMetroRouteProvider

Builds an adjacency graph from service-pattern station pairs, searches it with a minimum-cost heap, and assembles the winning edges into ride and transfer segments.

COVERAGE GATEBoth endpoints within 25 km of network bounds.
ACCESS CANDIDATESFour nearest stations for each endpoint.
GENERATED PATHSFastest, fewest transfers, least walking; duplicate edge sequences are removed.
TRANSFER COST300 seconds normally; 1,200 seconds for fewest-transfer search.
WALK COSTDistance ÷ 1.25 m/s; multiplied by three in least-walking search.
TRAIN COSTmax(60, edgeDistance ÷ 9.7 + 30) seconds.
WALKING LEGSMKDirections when available; straight-line + 1.25 m/s fallback.
GEOMETRY GUARDSHandles ring-line seams and rejects implausibly long polyline matches.

05 / CITY PACKS

Enrichment is downloaded per city.

OfficialCityPackService owns manifest lookup, pack downloads, validation, in-memory indexing, and station enrichment.

01Find a manifest

Try configured manifest/base/fallback values, then jsDelivr, then raw GitHub. Duplicate URLs are removed.

02Find the city

The manifest entry decides whether a pack is available, source-pending, unavailable, or not configured.

03Coalesce work

Concurrent callers for the same city share one in-flight task. A delete increments the load generation and cancels stale completion.

04Download & validate

Each request has a 15-second timeout. Decoded city ID and version must exactly match the manifest entry.

05Index in memory

Station records are keyed by normalized name. The service does not write an app-managed persistent pack file.

06Enrich on demand

Stations receive lines, accessibility, facilities, schedules, maps, timetable assets, service status, exits, platform hints, and interchange hints when fields exist.

Important operational details

  • A failed load is cached for 45 seconds, preventing every station lookup from restarting the full fallback chain while the rider is underground.
  • Manifests are cached by URL for the app session. Loaded packs remain decoded in memory until deletion or memory-pressure release.
  • Exit guidance prefers structured pack entries. If none exist, a short exit token may be extracted from accessibility text and labeled estimated.
  • “Train times” are represented as official schedule rows. Their remaining-minutes field is nil unless a future live provider supplies it.

07 / CONFIDENCE SERVICES

Pure calculations are kept separate.

RouteFeasibilityService

Combines step-free state, route warnings, walking instruction keywords, local reports, and comfort. It produces good, caution, risky, or unknown; explanations; a bottleneck; and estimated extra minutes.

RouteConfidenceService

Applies the documented 0–100 rule set after feasibility is known. It returns a level, one mode-aware explanation, positive reasons, and warnings. It performs no network calls and uses no machine-learning model.

ServiceHoursResolver

Parses first/last schedule strings, including multiple times separated by slash and last trains after midnight. “Last train soon” means within 20 minutes of the latest matched last-train time.

ComfortForecastService

Only runs when source-backed crowd-control windows exist for the route. An active official window produces busy; otherwise weekday 07:30–09:30 or 17:00–19:30 can produce moderate; outside those periods the route is labeled likely comfortable.

LiveGoTripBuilder

Turns route segments into ordered walking, transfer, ride, and arrival steps. It uses only the selected route snapshot and carries arrival-exit hints into ride steps.

08 / STATION SERVICES

Search local first, then broaden.

StationSearchService combines the selected city’s bundled station list with Apple place search and city-pack enrichment.

  • An empty keyword returns the bundled city station list.
  • A keyword first filters station name, English name, and pinyin locally.
  • Apple place results are resolved concurrently; a result matching a known station is replaced by the station model.
  • Results are deduplicated by city plus normalized station name.
  • Filters can require known full accessibility, a known elevator, transfer status, or a facility type.
  • When location exists, filtered station results are sorted by calculated distance; otherwise they are sorted by localized name.

10 / NOTIFICATIONS

Local reminders, requested lazily.

TripReminderService is the only notification layer in the current source.

  • Authorization is requested when the rider first tries to schedule a reminder, never automatically at launch.
  • Departure reminders use a nonrepeating calendar trigger at the calculated leave-by time minus the selected lead minutes.
  • Get-off reminders use a nonrepeating time-interval trigger based on estimated route-segment duration.
  • Past-dated reminders are not scheduled. Replacing a reminder cancels the previous request with the same route or step identifier.
  • Foreground reminders are explicitly shown as a banner with sound.
  • No remote push provider, device token, or notification analytics appears in the service.

11 / CACHING & FAILURE

Fail fast enough to remain usable underground.

ConditionBehavior
Stalled MapKit callThe app-side 12-second race releases the caller and loading UI. The Apple framework request may continue in the background, so stale-result guards remain necessary.
Stalled pack hostThe URL request has a 15-second timeout before the next fallback host is tried.
Repeated pack failureThe city returns the cached failed status for 45 seconds instead of restarting every enrichment request.
Concurrent pack requestsOne in-flight task is shared per city. A later deletion invalidates its generation.
Malformed networkThe city is marked missing for the session rather than crashing the route or map flow.
Missing walking routeUse straight-line distance and estimated time; attach an explicit estimate note.
Memory warningRelease decoded city packs, full metro networks, station projections, graph caches, and normalized-name indexes.
App termination during GoRetain the active route in UserDefaults and offer a resume banner on the next planner appearance.
